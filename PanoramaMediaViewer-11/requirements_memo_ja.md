# 要件メモ

## 一覧画面

- ファイルアプリ上の任意フォルダから読み込む
- 写真アプリからの読み込みは実装しない
- 写真と動画は分けない
- サムネイルは表示しない
- 拡張子はデフォルト非表示
- 先頭に文字で種別を表示する
  - `写真`
  - `動画`
- 表示例
  - `写真 THETA_0001`
  - `動画 THETA_0002`
- 撮影日時順ソートは実装しない
- 検索欄は実装しない
- 画面横幅が許す限り複数列で表示する
- Split View対応は必須要件から外す
- 原則としてフルスクリーン利用を想定する
- 誤タップを減らすため、項目全体をタップ対象にする

## 閲覧画面

- タップした360度写真または360度動画を表示
- 指でドラッグして視点を回転
- ピンチでズーム
- 動画は再生・一時停止
- 動画はシークバーで任意の時間へ移動できる
- 動画シークバーをドラッグ中も映像更新を試みる

## 対象データ

- ファイルアプリ内の任意フォルダ直下の写真・動画
- 正距円筒投影の360度写真・360度動画

## 2026-05-31 update v2

- 360ビューアのドラッグ操作方向を反転。
- 動画表示中、スワイプしていない状態でも映像が更新されるように、SceneKitビューを継続描画する設定を追加。
- 動画用コントロールにシークバーを追加。
- 動画の現在時刻 / 総時間を表示。

## 2026-05-31 update v3

- 写真アプリからの取り込み機能を廃止。
- 読み込み元をファイルアプリ上のフォルダのみに整理。
- 写真ライブラリ権限説明をInfo.plistから削除。
- シークバーをドラッグしている間も、AVPlayerのseekを実行して映像追従を試みるように変更。


## v5 変更

- フォルダ内の写真・動画一覧をファイル名順で並べ替える。
- 拡張子は表示しないが、表示名を基準に自然順で並べる。


## v6 変更

- Split View対応を必須要件から外した。
- `Info.plist` の `UIRequiresFullScreen` を `true` に変更した。
- フォルダ選択時のSecurity Scoped Resourceを、再選択時・解放時に停止するように整理した。

## v7 追加要件

- iPhoneにも対応する。
- iPadにも引き続き対応する。
- Split Viewには対応しない。
- iPhoneの狭い画面では、一覧は自動的に1列または少数列に縮退する。

## v8 変更

- 動画再生時の下部コントロールを2行から1行に圧縮。
- 再生 / 一時停止ボタン、シークバー、再生時間表示を1行にまとめた。
- 映像面積を広げるため、コントロールは画面下部に小さく重ねて表示する。
- 再生 / 一時停止は文字ボタンではなく、コンパクトなアイコンボタンに変更した。


## v9 追加修正

- 動画コントロールを画面最下部に寄せる。
- ビューア画面はナビゲーションバーを非表示にし、写真・動画の表示領域を最大化する。
- iPhone表示時の上下空白を減らすため、360ビューアを全画面表示にする。
- 戻る操作は画面左上の小さな戻るボタンで行う。
